What is HTML?

HTML, which stands for Hypertext Markup Language, is the standard language used for creating and designing the structure of a web page. It allows you to organize content on your website, define its structure, and establish the relationships between different elements.

Basic Structure of an HTML Document

An HTML document follows a specific structure that acts as the foundation for your web page:


    <!DOCTYPE html>
    <html lang="en">
    <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
    </head>
    <body>
        <!-- your web page content goes here -->
    </body>
    </html>

Let's break it down:

<!DOCTYPE html>: defines the document type and version of HTML being used (HTML5 in this case).

<html lang="en"> and </html>: opening and closing tag of the root element that wraps the entire HTML content. The attribute lang="en" defines the language (in this case, USA English).

<head> and </head>: opening and closing tag of the head element contains meta-information <meta> about the HTML document, the page title <title></title> you see in the browser tab, and link <link /> which defines a link between your HTML document and an external resources, like stylesheet, favicon, import and so on.

<body> and </body>: opening and closing body tag encloses all the visible content of a web page, including text, images, links, forms, and other elements that users interact with.

Note: All HTML elements have opening (< >) and closing (</ >) tags, except for self-closing (< > or < />) tags, which I will explain in more detail later.

Comments

Notice this <!-- your web page content goes here --> in the above HTML basic structure, it's called comments. Comments are used to add explanatory notes that are not displayed when the web page is viewed in a browser. They are useful for documenting your code, providing information to other developers, or temporarily excluding specific parts of the code. You can create a comment using this tag <!-- your comment goes here -->.

There are:

  1. Single-line comment: <!-- This is a single-line comment -->

  2. Multi-line comment: <!-- This is a multi-line comment. It can span multiple lines.All content within the comment block will be ignored by the browser. -->

Tags and Elements

HTML uses tags to define different elements on a webpage. Tags are enclosed in angle brackets (< >). There are opening (<tag>) and closing (</tag>) tags, and self-closing tags.

Headings


    <h1>This is a Heading 1</h1>
    <h2>This is a Heading 2</h2>

The heading tags <h1> to <h6> are used to define headings or subheadings within a document. These tags represent a hierarchy of headings, with <h1> being the highest level (main heading) and <h6> being the lowest level (lowest subheading level).

Its purpose is to structure and organize the content of a web page, making it more readable and accessible.

Paragraph

The paragraph tag (<p>) is used to separate blocks of text into distinct paragraphs. It is a block-level element that represents a unit of text or a block of content, and it is commonly used to structure and separate text on a webpage.

Line Breaks

To create a line break without starting a new paragraph, use the break (<br>) tag.

Example 1 - Basic Line Break:


    <p>This is the first line.<br>This is the second line.</p>

This will render as:

This is the first line.
This is the second line.

Div

A <div> tag, which stands for "division," is one of the most commonly used container elements in HTML. It is a block-level container that is used to group other HTML elements together and apply styles or scripting to them collectively.

Here's an example:


    <div>
        <p>This is a paragraph inside a div.</p>
        <ul>
            <li>List item 1</li>
            <li>List item 2</li>
        </ul>
    </div>

Semantic Tags

Semantic tags are like special labels that tell web browsers and developers what different parts of a webpage are all about. They help make websites easier to understand for both people and computers.

By using these tags, you can make your websites more accessible and easier to find on search engines. Here are some common HTML semantic tags along with examples:

  1. <header>: Represents introductory content at the beginning of a section or webpage.
  2. <nav>: Defines navigation links within your webpage.
  3. <main>: Defines the main content of a webpage.
  4. <section>: Defines sections within a webpage.
  5. <article>: Defines an independent piece of content that can stand alone.
  6. <aside>: Defines content related to the main content but not part of it.
  7. <footer>: Defines the footer of a webpage.

List Tag

Lists allow you to organize and structure content in a hierarchical manner. There are two types: ordered (<ol>) and unordered (<ul>) lists.

Example of an ordered list:


    <ol>
        <li>First item</li>
        <li>Second item</li>
        <li>Third item</li>
    </ol>

Example of an unordered list:


    <ul>
        <li>Item 1</li>
        <li>Item 2</li>
        <li>Item 3</li>
    </ul>

Span Tag

The <span> tag is a generic inline container used to group and apply styles to inline elements or text. It is typically used when you want to apply styles or manipulate specific portions of text within a larger block of content.

Example:


    <p>This is a <span style="color: red; font-weight: bold;">highlighted</span> text.</p>

Links

The <a> tag is used to create hyperlinks or anchor points within a webpage. It is primarily used to define a hyperlink, allowing users to navigate to another webpage, a different section of the same page, or even an external resource.

Example:


    <a href="https://www.example.com">Visit Example.com</a>

Form Tag

HTML forms are essential for user interaction on websites. They allow users to input data that can be sent to a server for processing. The basic structure of an HTML form involves several key elements:


    <form>
        <label for="username">Username:</label>
        <input type="text" id="username" name="username" placeholder="Enter your username">
        <button type="submit">Submit</button>
    </form>

HTML Attributes

Attributes provide additional information about HTML elements. They are always included in the opening tag and are written as name-value pairs.

Example:


    <img src="image.jpg" alt="Description of the image" width="200" height="150">

HTML Multimedia

You can integrate various types of media content into web pages, such as images, audio, and video.

Example of embedding an image:


    <img src="image.jpg" alt="Description of the image" width="200" height="150">

Example of embedding audio:


    <audio controls>
        <source src="audio.mp3" type="audio/mpeg">
        Your browser may not support the audio element.
    </audio>

Example of embedding video:


    <video controls width="640" height="360">
        <source src="video.mp4" type="video/mp4">
        Your browser may not support the video element.
    </video>

Best Practices

  1. Follow proper HTML document structure.
  2. Use semantic HTML elements.
  3. Comment your code.
  4. Structure your content with proper tags.
  5. Group elements with <div> and <span> sparingly.
  6. Do not overuse line breaks (<br>).
  7. Always use alternative text for images.
  8. Optimize forms for user experience.
  9. Ensure compatibility with older browsers.
  10. Stay updated with HTML standards.

HTML Meta Tags

Meta tags provide metadata about an HTML document. They are placed inside the <head> section of the document and help improve the functionality of the page in search engines, social media, and browsers.

Common meta tags include:

  • <meta charset="UTF-8">: Specifies the character encoding for the document.
  • <meta name="description" content="Your page description">: Provides a description of the page for search engines and social sharing platforms.
  • <meta name="keywords" content="HTML, web development, coding">: Specifies keywords relevant to the page for search engine optimization (SEO).
  • <meta name="author" content="Your Name">: Indicates the author of the webpage.
  • <meta http-equiv="X-UA-Compatible" content="IE=edge">: Ensures the page is displayed correctly on Internet Explorer.

Forms and Input Elements

HTML forms allow users to input data that can be sent to the server. A form usually contains various types of <input> elements such as text fields, buttons, and checkboxes.

The most common form elements include:

  • <input type="text">: Used for single-line text input fields.
  • <input type="password">: Used for password fields.
  • <input type="email">: Used for input fields that require an email address.
  • <textarea>: Used for multi-line text input.
  • <button>: Used for clickable buttons that submit forms.
  • <select> and <option>: Used to create dropdown lists.

Hereâ€™s an example of a form with various input types:


    <form action="submit_form.php" method="POST">
        <label for="name">Name:</label>
        <input type="text" id="name" name="name" placeholder="Your name">
            
        <label for="email">Email:</label>
        <input type="email" id="email" name="email" placeholder="Your email">
            
        <label for="message">Message:</label>
        <textarea id="message" name="message" placeholder="Your message"></textarea>
            
        <button type="submit">Submit</button>
    </form>

HTML Tables

HTML tables are used to represent data in a tabular format, consisting of rows and columns. A table is created using the <table> element, and rows are created with the <tr> element. Cells within the rows are created using <td> for data cells and <th> for header cells.

Hereâ€™s an example of a simple HTML table:


    <table border="1">
        <tr>
            <th>Name</th>
            <th>Age</th>
        </tr>
        <tr>
            <td>John Doe</td>
            <td>25</td>
        </tr>
        <tr>
            <td>Jane Smith</td>
            <td>30</td>
        </tr>
    </table>

Attributes you can use with tables:

  • border: Adds a border around the table.
  • cellspacing: Defines space between cells.
  • cellpadding: Defines space within cells.

Form Elements and Their Attributes

HTML forms are essential for getting user input. They can contain several different types of form controls such as text fields, buttons, checkboxes, radio buttons, and more. Hereâ€™s a breakdown of some commonly used form controls and their attributes:

Radio Buttons and Checkboxes

Radio buttons allow a user to select only one option from a set, while checkboxes allow users to select multiple options. Both elements are used with the <input> tag.


    <form>
        <label>Do you like HTML?</label><br>
        <input type="radio" id="yes" name="html" value="yes"> Yes<br>

        <input type="checkbox" id="green" name="colors" value="green"> Green<br>

Advanced HTML Elements

Once youâ€™re comfortable with the basics of HTML, there are a variety of advanced elements you can use to enhance your webpages:

IFrames

An <iframe> (inline frame) is used to embed another document within the current document. This is typically used for embedding external content such as videos, maps, or other websites.


    <iframe src="https://www.example.com" width="600" height="400"></iframe>

Embedding Other Media

You can embed external media, such as audio or video, directly within your page using the <embed> tag. This is particularly useful for embedding Flash or other plugin-based content.


    <embed src="flash_video.swf" width="600" height="400">

Custom Styling with Classes and IDs

Using the class and id attributes, you can add custom styles to specific elements on your webpage. class is used for styling multiple elements, while id is used to uniquely identify a single element.


    <div class="custom-style">
        <p>This is a paragraph with custom styling.</p>
    </div>
            
    <div id="unique-element">
        <p>This is a uniquely identified element.</p>
    </div>

Best Practices for HTML Forms

Here are some tips for creating user-friendly forms:

  • Always include clear labels for each form field.
  • Group related fields together to make the form more organized.
  • Use placeholder text to provide examples or hints inside input fields.
  • Validate form inputs before submission to avoid errors.
  • Ensure forms are accessible by adding aria-label attributes for screen readers.
  • Provide feedback to the user once the form is submitted successfully or if errors occur.

Viewport and Responsive Web Design

Responsive web design ensures that a webpage looks good on all devices, from desktops to smartphones. You can make a webpage responsive by using the <meta name="viewport"> tag in the <head> section. The viewport meta tag controls the layout on mobile browsers, ensuring your content scales correctly on smaller screens.


    <meta name="viewport" content="width=device-width, initial-scale=1.0">

Embedding External Content (iFrames)

iFrames are used to embed other web content (such as videos, external web pages, or maps) inside your webpage.


    <iframe src="https://www.youtube.com/embed/example" width="560" height="315"></iframe>

HTML Text Formatting

HTML provides several elements for formatting text:

  • <b>: Defines bold text.
  • <i>: Defines italicized text.
  • <u>: Defines underlined text.
  • <small>: Defines smaller text.
  • <mark>: Highlights text.
  • <del>: Defines deleted (struck-through) text.
  • <ins>: Defines inserted text (underlined).

Best Practices for Comments

HTML comments are important for code clarity, but they should be used sparingly and never for leaving TODOs or non-descriptive notes.

  • Always write comments to explain sections of code that might be complex or non-obvious.
  • Avoid cluttering your code with too many comments.
  • Remove commented-out code before deploying to production.

Form Validation

To ensure user inputs are correct, HTML5 provides several form validation attributes such as required, pattern, minlength, maxlength, etc. These attributes allow you to perform client-side validation before form submission.

Example:


    <form>
        <input type="email" name="email" required>
        <input type="text" name="name" minlength="3" required>
    </form>
